# 基于"缝"的构件编码体系

## 编码层级

1. 槽缝：朝向+内外，如 `CAO_s-o`（南面外槽）、`CAO_s-i`（南面内槽）
2. 栿缝 / 间缝：位置，如 `FU_w1`（西一间缝）
3. 槫缝 / 橑檐枋缝：朝向+位置，如 `TUAN_m`（脊槫缝）、`TUAN_s1`（上平槫缝）
4. 铺作缝（含柱缝）：所属槽缝+位置，如 `PUZUO_s-o-m`（南面外槽中线）、`PUZUO_s-o-w1`（南面外槽西侧第一朵铺作缝）、`PUZUO_sw-o`（西南转角外槽）
5. 跳心缝：所属铺作缝+内外位置，如 `TIAO_s-o-w1-m`（南面外槽西侧第一朵铺作中线）、`TIAO_s-o-w1-o1`（南面外槽西侧第一朵铺作外跳第一跳）

## 基本属性

### 规则

- 有共识性的符号小写（为区分L和1，L采用大写），特殊符号大写简称（可根据预设自定义增加和修改，参考Rhino）。

### 特点

- 影响力越大的槽编码越短

| 类型     | 描述                                                                                                |
| ------ | ------------------------------------------------------------------------------------------------- |
| 朝向     | s：南；n：北；w：西；e：东；m：中                                                                               |
| 内外     | i：内；o：外                                                                                           |
| 位置     | 0+：距离中线的位置                                                                                        |
| 缝类型标识  | CAO：槽缝；FU：栿缝；TUAN：槫缝 / 橑檐枋缝；PUZUO：铺作缝；TIAO：跳心缝                                                    |

## 对应构件——以《营造法式》五铺作为例

### 通用规则

- 增加层。
- 构件默认方向为与面阔的相对角度（类似法线），转角斗栱的面阔方向是斜向45°。
- 分支做法标注在开头（构造实体大写，做法小写，如ANZHI（暗栔）、dancai（单材），整体思路为：**做法_位置**，因此不存在宋式和清式的转化问题，因为本质上为不同做法，而替换构件时可以选择任何（合理或不合理）的构件。
- 每个构件自带默认榫卯（在拓扑不变的情况下可以调整数值），也许未来可以考虑以榫卯作为编码开头的后缀，如GUAZIGONG-ZIYIN之类的。
- 每个构件的位置标示的是其核心参考位置（因此下昂可以调整编码后缀以从属于不同参考点），其余各节点皆为相对于这一位置的参考点，如挪动则需要改变参考位置，或改变相对于参考位置的相对位置（偏移量）。
- 柱头枋等联系件的部分属性可拥有双位置信息，如mw1、w1w2，通过识别可从属于两个铺作；同理，位于交叉位置的构件可以拥有双位置信息，如sw。与转角铺作发生关系的联系件属于转角铺作，以方向进一步区分两个构件，可改变参考位置以使其属于相邻的补间铺作。
- 昂栓等位于缝之间的构件可以用o.5、i.7的方式表示与缝之间相对位置的百分比。

## 反馈

1. 朝向：前后左右
2. 转角：三条有方向性的缝
3. 榫卯属于多个构件
